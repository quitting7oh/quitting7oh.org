# Structures to Avoid

## Binary Contrasts

These create false drama. State the point directly.

| Pattern | Problem |
|---------|---------|
| "Not because X. Because Y." / "Not because X, but because Y." | Telegraphed reversal |
| "[X] isn't the problem. [Y] is." | Formulaic reframe |
| "The answer isn't X. It's Y." | Predictable pivot |
| "It feels like X. It's actually Y." | Setup/reveal cliche |
| "The question isn't X. It's Y." | Rhetorical misdirection |
| "Not X. But Y." / "not X, it's Y" / "isn't X, it's Y" | Mechanical contrast |
| "It's not this. It's that." | Same formula, different words |
| "stops being X and starts being Y" | False transformation arc |
| "doesn't mean X, but actually Y" | Negation-then-assertion crutch |
| "is about X but not Y" | False distinction |
| "not just X but also Y" | Additive hedge |

**Instead:** State Y directly. "The problem is Y." "Y matters here." Drop the negation entirely.

## Negative Listing

Listing what something is *not* before revealing what it *is*. A rhetorical striptease.

| Pattern | Problem |
|---------|---------|
| "Not a X... Not a Y... A Z." | Dramatic buildup through negation |
| "It wasn't X. It wasn't Y. It was Z." | Same structure, past tense |

**Instead:** State Z. The reader doesn't need the runway.

## Dramatic Fragmentation

Sentence fragments for emphasis read as manufactured profundity.

| Pattern | Problem |
|---------|---------|
| "[Noun]. That's it. That's the [thing]." | Performative simplicity |
| "X. And Y. And Z." | Staccato drama |
| "This unlocks something. [Word]." | Artificial revelation |

**Instead:** Complete sentences. Trust content over presentation.

## Rhetorical Setups

These announce insight rather than deliver it.

| Pattern | Problem |
|---------|---------|
| "What if [reframe]?" | Socratic posturing |
| "Here's what I mean:" | Redundant preview |
| "Think about it:" | Condescending prompt |
| "And that's okay." | Unnecessary permission |

**Instead:** Make the point. Let readers draw conclusions.

## Self-Posed Rhetorical Questions

The model asks a question nobody asked, then answers it immediately for dramatic effect.

| Pattern | Problem |
|---------|---------|
| "The result? Devastating." | Manufactured suspense |
| "The worst part? Nobody saw it coming." | Same formula |
| "The scary part? It works." | Same |
| "The catch? You can't undo it." | Same |

**Fix:** Fold the question and answer into a statement. "The result is devastating." Or skip the framing and lead with the consequence.

## Anaphora Abuse

Repeating the same sentence opening across consecutive sentences. AI uses this for false momentum.

| Pattern | Problem |
|---------|---------|
| "They assume X. They assume Y. They assume Z." | Mechanical repetition |
| "They could expose X. They could offer Y. They could provide Z." | List disguised as prose |
| "They have built X, but not Y. They have built A, but not B." | Parallel stacking |

**Fix:** Combine related points into one sentence. Or if the list is genuinely a list, present it as a list.

## Listicle in a Trench Coat

Numbered or labeled points dressed up as continuous prose. The model writes a listicle but wraps each point in a paragraph starting with "The first... The second... The third..." to disguise the format.

| Pattern | Problem |
|---------|---------|
| "The first wall is X. The second wall is Y. The third wall is Z." | Numbered list pretending to be prose |
| "The second takeaway is... The third takeaway is..." | Same formula |
| "Step one is... Step two is... Step three is..." | Same |

**Fix:** If the content is a list, format it as a list. If it should be prose, weave the points together without numbering.

## Superficial Participle Analyses

Tacking a present-participle (`-ing`) phrase onto the end of a sentence to inject shallow analysis. The model assigns significance, legacy, or broader meaning to mundane facts.

| Pattern | Problem |
|---------|---------|
| "contributing to the region's rich cultural heritage" | Hollow significance-signaling |
| "highlighting its enduring legacy" | Same |
| "underscoring its role as a dynamic hub" | Same |
| "reflecting broader trends in [field]" | Same |
| "demonstrating the importance of [topic]" | Same |
| "marking a turning point in [area]" | Same |

**Fix:** Either make a specific analytical claim or delete the participle phrase. If you can't say *what* it underscores, "underscoring" is empty.

## False Ranges

"From X to Y" constructions where X and Y are not on any real scale. In legitimate use, "from X to Y" implies a spectrum with a meaningful middle. AI uses it to dress up a two-item list.

| Pattern | Problem |
|---------|---------|
| "From innovation to implementation to cultural transformation." | No real spectrum, no middle |
| "From the small startup to the global enterprise..." | Just "all sizes" with fancy framing |
| "From problem-solving to scientific discovery to artistic expression." | Fancy list, not a range |

**Fix:** If the items are a list, list them. If there's a real spectrum, describe it (and what falls in between).

## "Despite Its Challenges..."

AI acknowledges problems only to immediately dismiss them. The setup-and-pivot is mechanical.

| Pattern | Problem |
|---------|---------|
| "Despite these challenges, the initiative continues to thrive." | Formulaic optimism |
| "Despite its prosperity, [X] faces challenges typical of urban areas." | Structured dismiss-and-pivot |
| "While X has limitations, it remains a promising approach." | Same beat, different wording |

**Fix:** If the challenges matter, analyze them. If they don't matter, skip them. Don't acknowledge-then-pivot.

## Invented Concept Labels

AI invents compound labels that sound analytical without being defined. It appends abstract problem-nouns (paradox, trap, creep, divide, vacuum, inversion, asymmetry) to domain words and uses them as if they were established terms.

| Pattern | Problem |
|---------|---------|
| "the supervision paradox" | Invented term treated as established |
| "the acceleration trap" | Same |
| "workload creep" | Same |
| "the expertise vacuum" | Same |
| "the X-Y divide" | Same |

**Fix:** If the concept needs a name, define it the first time and use it sparingly. If it doesn't need a name, describe the dynamic in plain language. Multiple invented labels in the same piece is a strong AI signal.

## One-Point Dilution

Making a single argument and restating it in ten different ways. The model pads a simple thesis to feel "comprehensive" by rephrasing with different metaphors, examples, and framings.

| Symptom | Problem |
|---------|---------|
| Same claim restated across 4000 words with different metaphors | Padding for length |
| Each section rephrases the thesis instead of advancing it | No new ground |
| The piece could be 800 words and lose nothing | Excessive elaboration |

**Fix:** State the point once, support it concretely, move on. If a section circles back to the same claim, cut the repetition.

## The Dead Metaphor

Latching onto a single metaphor and using it in every paragraph. A human writer introduces a metaphor, uses it once or twice, and moves on. AI repeats it 5-10 times.

| Pattern | Problem |
|---------|---------|
| "Walls and doors" used 30+ times in one article | Beating the metaphor into the ground |
| Every paragraph finds a way to mention "primitives" | Same |
| The "ecosystem" metaphor structures every section | Same |

**Fix:** Use a metaphor once or twice. Then drop it. Find a different image, or just describe the thing.

## Formulaic Constructions

| Pattern | Problem |
|---------|---------|
| "By the time X, I was Y." | Narrative template |
| "X that isn't Y" | Indirect. Say "X is broken" |

## False Agency

Giving inanimate things human verbs. Complaints don't "become" fixes. Bets don't "live or die." Decisions don't "emerge." A person does something to make those things happen. AI loves this because it avoids naming the actor.

| Pattern | Problem |
|---------|---------|
| "a complaint becomes a fix" | The complaint did nothing. Someone fixed it. |
| "a bet lives or dies in days" | Bets don't have lifespans. Someone kills the project or ships it. |
| "the decision emerges" | Decisions don't emerge. Someone decides. |
| "the culture shifts" | Cultures don't shift on their own. People change behavior. |
| "the conversation moves toward" | Conversations don't move. Someone steers. |
| "the data tells us" | Data sits there. Someone reads it and draws a conclusion. |
| "the market rewards" | Markets don't reward. Buyers pay for things. |

**Instead:** Name the human. "The team fixed it that week" beats "the complaint becomes a fix." If no specific person fits, use "you" to put the reader in the seat.

## Narrator-from-a-Distance

Floating above the scene instead of putting the reader in it.

| Pattern | Problem |
|---------|---------|
| "Nobody designed this." | Disembodied observation |
| "This happens because..." | Lecturer voice |
| "This is why..." | Same |
| "People tend to..." | Armchair sociologist |

**Instead:** Put the reader in the room. "You don't sit down one day and decide to..." beats "Nobody designed this."

## Passive Voice

Every sentence needs a subject doing something. Passive voice hides the actor and drains energy.

| Pattern | Fix |
|---------|-----|
| "X was created" | Name who created it |
| "It is believed that" | Name who believes it |
| "Mistakes were made" | Name who made them |
| "The decision was reached" | Name who decided |

**Instead:** Find the actor. Put them at the front of the sentence.

## Sentence Starters to Avoid

| Pattern | Fix |
|---------|-----|
| Sentences starting with What, When, Where, Which, Who, Why, How | Restructure. Lead with the subject or the verb. |
| Paragraphs starting with "So" | Start with content |
| Sentences starting with "Look," | Remove |

Wh- openers become a crutch. "What makes this hard is..." becomes "The constraint is..." or better, name the specific constraint.

## Headers

Headers fall into the same trap as sentence openers, more visibly because they're load-bearing scaffolding readers scan first.

| Pattern | Fix |
|---------|-----|
| "What helps" / "What this means" / "What to expect" | Lead with the noun. "Tools that help." "The expectations." Or use a subject-led header. |
| "Why this matters" / "Why X happens" | Make it a statement. "The reasons." "Mechanism." |
| "When to call for help" / "When to start" | Acceptable as a time-marker; less Wh-shaped than the rest. If overused, vary. |
| "How X works" / "How to Y" | "Mechanism." "Steps." Or restructure as a noun phrase. |
| "Does X actually do Y?" | Make a statement, not a rhetorical question. |
| "Is this normal?" | Same. State the answer directly. |

A page where every section header is a question reads as a FAQ that doesn't know it. Use questions sparingly — once or twice on a page where the reader *is* asking that question. Otherwise, headers are nouns or short verb phrases.

## Symmetric Structures

AI loves balance. Real human writing is asymmetric.

| Pattern | Problem |
|---------|---------|
| Pros / Cons lists with exactly the same item count | The world doesn't sort that way. One side usually has more. |
| "In favor:" / "Against:" tables with equal-length bullets | Manufactured balance. |
| Three-bullet groups with identical sentence shape | Each bullet starts the same way, runs the same length, uses the same construction. |
| Parallel-clause sentences where every member is the same shape | "It's not X, not Y, not Z. It's W." (Negative listing plus symmetric construction.) |

**Test:** if you can swap two bullets without changing meaning, suspect the list. If both sides of a contrast have the same number of items, suspect the contrast.

**Fix:** ship asymmetric. Three items here, two there, one in the third spot. Different sentence lengths within a list. One bullet that's a full sentence; another that's a fragment.

## Rhythm Patterns

| Pattern | Fix |
|---------|-----|
| Three-item lists | Use two items or one |
| Questions answered immediately | Let questions breathe or cut them |
| Every paragraph ends punchily | Vary endings |
| Em-dashes | Remove. The fix is usually **two sentences** (a period), occasionally a comma. Don't swap em-dash for a colon or parentheses. That keeps the same em-dash flow with different punctuation and reads identically slop. |
| Staccato fragmentation | Don't stack short punchy sentences |
| "Not always. Not perfectly." | Hedging disguised as reassurance |

## Word Patterns

| Pattern | Problem |
|---------|---------|
| Lazy extremes (every, always, never, everyone, everybody, nobody) | False authority. Use specifics instead of sweeping claims. |
| All adverbs (-ly words, "really," "just," "literally," "genuinely," "honestly," "simply," "actually") | Empty emphasis. See phrases.md for full list. |
